# トラブル時のテンプレ

キッティングが途中で止まった時、Claude（CC）に貼り付けるためのテンプレ集。

---

## Claude に貼るときのコピペ用テンプレ

```
MSA キッティングで失敗しました。

【状況】
- 何のステップで失敗したか: [例: アプリ一括インストール (winget)]
- いつ失敗したか: [今日の日時]
- 使っている PC: [岡島さんから渡された端末番号]

【ログ全文】
（C:\msa-kitting-logs\setup-YYYYMMDD-HHMMSS.log の内容を Ctrl+A → Ctrl+C で全選択コピーして貼り付け）
```

ログファイルが大きすぎて貼れない場合は、画面に出ていた **赤い [ERROR] の行** とその前後 20 行ほどを貼れば OK。

---

## よくあるエラーと対処

### 1. `winget が見つかりません`

```
[ERROR] winget が見つかりません
```

**原因**: Windows 11 でも初期状態だと winget が古いことがある。

**対処**: Microsoft Store を開いて「アプリ インストーラー」を最新化してから再実行。

### 2. `インターネット接続に失敗`

```
[ERROR] リポジトリ取得に失敗: Invoke-WebRequest ...
```

**原因**: Wi-Fi が繋がっていない or プロキシ越し。

**対処**: 
- Edge ブラウザで `https://github.com` が開けるか確認
- 開けないなら Wi-Fi 再接続、それでもダメなら岡島さんに連絡

### 3. `アクセスが拒否されました`

```
Access to the path '...' is denied.
```

**原因**: PowerShell が管理者権限で起動されていない。

**対処**: PowerShell を閉じて、もう一度「管理者として実行」で起動し直す。

### 4. 既にインストール済のアプリでスキップしたい

```
[SKIP] 既にインストール済
```

これは **正常動作**。エラーではないのでそのまま進んで OK。

### 5. 途中で再起動が必要と言われた

再起動して、もう一度同じコマンドを貼って実行すれば続きから流れる（既に終わったステップは自動スキップされる）。

## CC への引き継ぎ判断基準

次のいずれかに当てはまったら、自分で頑張らず CC（Claude チャット）に投げる:

- ✅ 同じエラーで 2 回連続失敗した
- ✅ 画面のエラーメッセージの意味が分からない
- ✅ 30 分待っても画面が動かない
- ✅ ログファイルが作られていない（`C:\msa-kitting-logs\` が空）

CC は「もう一度同じコマンドを貼ってみて」「PowerShell 閉じて開き直して」みたいな指示を出すので、それに従えば OK。

## hide さん向け：失敗パターンの共有

社員が困ったログが返ってきたら、このリポジトリの GitHub Issue に貼って蓄積していくと、CC が次回以降のキッティングで先回りできる。

Issue テンプレ:
```
## 状況
端末: MSA-PC-XXX / Windows 11 Pro
日時: 2026-MM-DD HH:MM
ステップ: アプリ一括インストール

## ログ抜粋
[ERROR] ...

## 解決方法
（CC が出した解決コマンド or 手動対応内容）
```
